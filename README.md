-CEASER CIPHER TOOL-

-Overview-


This program implements a basic Caesar Cipher encryption and decryption tool. A Caesar Cipher is a simple substitution cipher that shifts each letter in a message by a fixed number of positions in the alphabet.
The program allows users to:
Encrypt plaintext messages using a shift value
Decrypt ciphertext using a shift value
Analyze letter frequency (if implemented)
Experiment with classical cryptography concepts
This project was created for educational purposes to demonstrate foundational cryptographic principles.

Check the pdf "EXPLAINING HOW IT WORKS" if you want more details.

How to Run the Program
Requirements
Processing IDE (https://processing.org/download)
Java (included with Processing)
Steps
1) Download or clone this repository.
2) Open the project folder in the Processing IDE.
3) Open the Ceaser_cipher.pde file.
4) Click the Run button in Processing.
Follow the on-screen instructions to encrypt or decrypt text.
If running from GitHub:
git clone https://github.com/AlexanderVal13/Ceaser_cipher.git
Then open the folder in Processing.

-Limitations-

This tool is for educational use only.
The Caesar Cipher is not secure by modern cryptographic standards.
It can be broken easily using brute force or frequency analysis.
It should NOT be used to protect passwords, personal data, or sensitive information.
It only shifts alphabetic characters and may not handle special characters robustly (depending on implementation).
This program demonstrates historical encryption methods, not real-world security systems.

-Ethical Considerations & Responsible Use-

This project is intended strictly for learning about classical cryptography and understanding how simple encryption systems work.
Because encryption tools can be misused, it is important to recognize:
Simple ciphers like this could be modified for obfuscation purposes.
Users unfamiliar with cryptography might incorrectly assume this provides real security.
Teaching weak encryption without context could give a false sense of protection.
For these reasons, this tool should only be used in educational settings, coursework, or experimentation. It is not a substitute for modern encryption standards such as AES, RSA, or other industry-grade cryptographic systems.
Responsible use means understanding both the power and the limitations of encryption technology.
